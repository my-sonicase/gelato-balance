Build a professional gelato recipe balancing web application. This is a standalone Next.js project that will eventually be merged into a larger education platform — so the design system must be identical to that platform (same fonts, colors, tokens). Read everything below before writing any code.

This prompt is based on a reference Italian gelato balancing software called "BilanciaLi — Bilanciatore di Gelati". Replicate its feature set, tab structure, ingredient groups, and calculation logic as described below.

---

## WHAT THIS IS

A professional tool used by gelato makers to balance recipes. "Balancing" means adjusting ingredient quantities until the mix hits precise targets for sugar content, fat, SLNG (milk solids non-fat), other solids, total solids, sweetening power (POD), and antifreeze power (PAC) — all simultaneously. Think of it like a chemistry worksheet that updates in real time.

There are three apps in this project (build in this order):
1. **Gelato Balancer** (Professional) — full balancing software, Phase 1, build this first
2. **Home Recipe App** (Simplified) — guided wizard, outputs a home recipe, Phase 2
3. **Pro Recipe App** (Simplified Professional) — guided wizard, outputs a professional recipe with full breakdown, Phase 2

This prompt covers Phase 1 only: the full Balancing Software.

---

## TECH STACK

- **Framework:** Next.js (App Router, TypeScript)
- **State management:** Zustand — the recipe builder has complex interdependent state (ingredient list, calculated totals, ingredient DB). Do not use React Context for this; use Zustand.
- **Styling:** Tailwind CSS + CSS custom properties (same design tokens as main platform — see Design System section)
- **Persistence:** localStorage only for Phase 1. No backend, no auth. Users can save/load recipes locally.
- **Charts:** Recharts — for the balance bar chart showing how close the recipe is to target ranges
- **Export:** browser `window.print()` with a print stylesheet. No PDF library needed.
- **Math:** pure TypeScript functions. All gelato calculations happen client-side. No server needed for calculations.
- **i18n:** next-intl — EN + IT from day 1, exactly like the main platform. Routes: `/en/balancer/*` and `/it/balancer/*`. Every UI string, label, tab name, parameter name, and error message must have both an English and Italian translation. No hardcoded strings in components — all go through `useTranslations()`.

Do NOT use: any server actions or API routes for calculations (pure client-side), any external calculation APIs.

## TAB 5 — CALCOLATORI (Calculators)

Three independent calculators for deriving PAC and other constants. These are tools for advanced users who want to add custom ingredients.

### Calculator 1 — PAC da Peso Molecolare (PAC from Molecular Weight)

**Purpose:** Calculate the PAC constant for any pure sugar molecule.

**Formula (Angelo Corvitto):**
```
PAC = 34,200 / molecular_weight
```

**Examples:**
- Sucrose: MW = 342 → PAC = 34,200 / 342 = **100**
- Dextrose: MW = 180 → PAC = 34,200 / 180 = **190**
- Fructose: MW = 122 → PAC = 34,200 / 122 ≈ **280**

**UI:** Single input for molecular weight (g/mol), displays calculated PAC. A reference table of common sugars and their molecular weights is shown below.

### Calculator 2 — PAC per Paste Grasse (PAC for Fatty Pastes)

**Purpose:** Calculate the net PAC contribution of pastes that contain both fat and sugar (e.g., pistachio paste, hazelnut paste, almond paste).

**Formula:**
```
PAC_pasta = (zuccheri_pct / 100) × PAC_zucchero − (grassi_vegetali_pct / 100) × K_grass
```
Where K_grass is a calibration constant for fat's antifreeze effect (approximately 0.43).

Fat in a paste reduces the effective PAC because fat doesn't contribute to freezing point depression.

**Example — Pasta di Pistacchio (49% fat, 6% sugar as sucrose):**
- Sugar contribution: (6/100) × 100 = 6.0
- Fat correction: (49/100) × 43 ≈ 21.07 (negative effect)
- Net PAC ≈ 6.0 − 21.07 = **−15.07** (the paste has a net negative PAC per 100g of paste)
- Display as negative value, meaning adding this paste slightly raises the freezing point

**UI:** Inputs for % grassi vegetali and % zuccheri (with sugar type selector), displays net PAC per 100g of paste.

### Calculator 3 — PAC per Coperture e Cioccolato (PAC for Chocolate and Coatings)

**Purpose:** Calculate the net PAC of chocolate products combining cocoa butter, dry cocoa, and sugar.

**Formula:**
```
PAC_cioccolato = (zucchero_pct / 100) × PAC_saccarosio
               + (cacao_secco_pct / 100) × PAC_cacao_secco
               − (burro_di_cacao_pct / 100) × K_grass
```
Where PAC_cacao_secco ≈ 0 (cocoa solids don't dissolve) and K_grass ≈ 43.

**Example — Dark chocolate 54% cocoa butter, 45% dry cocoa, 1% sugar:**
- PAC ≈ (1/100)×100 + 0 − (54/100)×43 ≈ 1 − 23.22 = **−22.22** per 100g (approx −128% as shown in reference)
- (Note: the reference shows PAC = −128.60% for a specific chocolate formulation — use this as validation)

**UI:** Inputs for % burro di cacao, % cacao secco, % zucchero. Displays net PAC per 100g of product.

## DOMAIN KNOWLEDGE — CALCULATIONS

### What the software calculates

Every gelato recipe is a mix of ingredients. Each ingredient contributes to these pools:
1. **Acqua (Water)** — determines texture and iciness. Calculated as 100% − all solids.
2. **Grassi (Fat)** — determines creaminess
3. **SLNG** (Solidi del Latte Non Grassi = MSNF) — milk proteins + lactose, affects structure
4. **Zuccheri (Sugars)** — total sugar content by type
5. **Altri Solidi** — fibers, stabilizers, cocoa solids, fruit solids, etc.
6. **Solidi Totali** = Grassi + SLNG + Zuccheri + Altri Solidi

### Core pool calculations

```typescript
// For each ingredient in the recipe:
const totalWeight = sum(ingredients.map(i => i.weightG))

const totalWaterG   = sum(ingredients.map(i => i.weightG * i.acquaPct / 100))
const totalGrassiG  = sum(ingredients.map(i => i.weightG * i.grassiPct / 100))
const totalSLNGG    = sum(ingredients.map(i => i.weightG * i.slngPct / 100))
const totalZuccheriG = sum(ingredients.map(i =>
  i.weightG * sum(Object.values(i.zuccheri)) / 100
))
const totalAltriSolidiG = totalWeight - totalWaterG - totalGrassiG - totalSLNGG - totalZuccheriG
const totalSolidiTotali = totalGrassiG + totalSLNGG + totalZuccheriG + totalAltriSolidiG

// Express as % of total weight:
const waterPct        = totalWaterG / totalWeight * 100
const grassiPct       = totalGrassiG / totalWeight * 100
const slngPct         = totalSLNGG / totalWeight * 100
const zuccheriPct     = totalZuccheriG / totalWeight * 100
const altriSolidiPct  = totalAltriSolidiG / totalWeight * 100
const solidiTotaliPct = totalSolidiTotali / totalWeight * 100
```

### POD calculation (Potere Dolcificante — Sweetening Power)

POD measures how sweet the recipe is relative to sucrose (sucrose = 100).

```typescript
// For each ingredient, for each sugar type in that ingredient:
let totalPOD_g = 0
for (const ingredient of ingredients) {
  for (const [sugarType, sugarPct] of Object.entries(ingredient.zuccheri)) {
    const sugarWeightG = ingredient.weightG * sugarPct / 100
    totalPOD_g += sugarWeightG * SUGAR_CONSTANTS[sugarType].POD / 100
  }
}
// Express as % of total mix weight:
const podPct = totalPOD_g / totalWeight * 100
```

### PAC calculation (Potere Anti-Congelante — Antifreeze Power)

PAC measures how much the sugars depress the freezing point. Higher PAC = softer gelato at serving temperature.

```typescript
// Same pattern as POD but using PAC constants:
let totalPAC_g = 0
for (const ingredient of ingredients) {
  for (const [sugarType, sugarPct] of Object.entries(ingredient.zuccheri)) {
    const sugarWeightG = ingredient.weightG * sugarPct / 100
    totalPAC_g += sugarWeightG * SUGAR_CONSTANTS[sugarType].PAC / 100
  }
}
const pacValue = totalPAC_g / totalWeight * 100
// PAC is reported as this value (not as a ratio to sucrose)
```

### Serving temperature calculation

```typescript
// Based on Angelo Corvitto's methodology:
const temperaturaServizio = -3.7 * (pacValue / 100)
// pacValue is already expressed as % of mix weight
// Example: PAC = 28.5% → T = -3.7 × 0.285 = -1.05°C
// Typical range: -9°C to -14°C for gelato
```

### Calorie calculation

```typescript
const fatKcal      = (totalGrassiG / totalWeight * 100) * 9
const sugarKcal    = (totalZuccheriG / totalWeight * 100) * 4
const proteinKcal  = (totalSLNGG / totalWeight * 100) * 0.35 * 4  // ~35% of SLNG is protein
const kcalPer100gMix = fatKcal + sugarKcal + proteinKcal
// Adjust for overrun (gelato is lighter than mix per 100g):
const kcalPer100gGelato = kcalPer100gMix / (1 + overrunPct / 100)
```

## DATA STRUCTURES (TypeScript types)

```typescript
// Sugar types as a union
type SugarType =
  | 'saccarosio' | 'destrosio' | 'fruttosio' | 'invertito' | 'lattosio'
  | 'miele' | 'trealosio' | 'isofruttosio' | 'maltitolo' | 'sorbitolo'
  | 'mannitolo' | 'xilitolo' | 'eritritolo' | 'tagatosio' | 'stevia'
  | 'glicrizina' | 'glucosioAt21DE' | 'glucosioAt39DE' | 'glucosioAt42DE'
  | 'glucosioAt52DE' | 'maltodestrina5DE' | 'maltodestrina18DE' | 'maltodestrina20DE'

// Profile types
type ProfileType =
  | 'gelato' | 'sorbetto' | 'granita' | 'vegan' | 'gastronomico'
  | 'personalizzato1' | 'personalizzato2'

// Ingredient group
type IngredientGroup =
  | 'latticiniUova' | 'neutriBasi' | 'zuccheri' | 'ingredientiPrincipali'
  | 'fruttaVerdura' | 'alcolici' | 'alimentiTritati'

// An ingredient in the database
interface IngredientDefinition {
  id: string
  nome: string            // Italian name
  nomeit?: string         // alternate Italian name
  group: IngredientGroup
  acquaPct: number        // grams per 100g
  grassiPct: number       // grams per 100g
  slngPct: number         // grams per 100g
  zuccheri: Partial<Record<SugarType, number>>  // grams per 100g for each sugar type
  altriSolidiPct: number  // auto-calculated: 100 - acqua - grassi - slng - sum(zuccheri)
  minPct?: number         // min % in total mix (ingredient-level constraint)
  maxPct?: number         // max % in total mix (ingredient-level constraint)
  isCustom?: boolean
  isReadOnly?: boolean    // built-in ingredients are read-only
}

// A line item in the recipe (ingredient + quantity)
interface RecipeLine {
  id: string
  ingredientId: string
  ingredient: IngredientDefinition
  weightG: number         // grams in this recipe
}

// Target ranges for one parameter in one profile
interface ParameterRange {
  min: number
  max: number
}

// Target ranges for a complete profile
interface ProfileRanges {
  zuccheri: ParameterRange
  grassi: ParameterRange
  slng: ParameterRange
  altriSolidi: ParameterRange
  solidiTotali: ParameterRange
  pod: ParameterRange
  pac: ParameterRange
  frutta: ParameterRange
  alcolici: ParameterRange
  overrun: ParameterRange
  alimentiTritati: ParameterRange
}

// A complete recipe
interface Recipe {
  id: string
  nome: string
  profile: ProfileType
  lines: RecipeLine[]
  overrunPct: number      // default: 35 for gelato/sorbetto/vegan/gastronomico, 0 for granita
  slotName?: string       // e.g. "Base bianca", "Mio Gelato 3"
  notes?: string
  createdAt: string
  updatedAt: string
}

// Calculated balance for the whole recipe
type BalanceStatus = 'low' | 'ok' | 'high'

interface RecipeBalance {
  totalWeightG: number
  acquaG: number;         acquaPct: number
  grassiG: number;        grassiPct: number;       grassiStatus: BalanceStatus
  slngG: number;          slngPct: number;         slngStatus: BalanceStatus
  zuccheriG: number;      zuccheriPct: number;     zuccheriStatus: BalanceStatus
  altriSolidiG: number;   altriSolidiPct: number;  altriSolidiStatus: BalanceStatus
  solidiTotaliG: number;  solidiTotaliPct: number; solidiTotaliStatus: BalanceStatus
  podValue: number;       podStatus: BalanceStatus
  pacValue: number;       pacStatus: BalanceStatus
  fruttaPct: number;      fruttaStatus: BalanceStatus
  alcoliciPct: number;    alcoliciStatus: BalanceStatus
  alimentiTritatiPct: number; alimentiTritatiStatus: BalanceStatus
  // DATI GELATO
  pesoMiscelaG: number    // = totalWeightG
  pesoGelatoG: number     // = totalWeightG × (1 + overrunPct/100)
  kcalPer100gGelato: number
  temperaturaServizio: number  // °C
  overallStatus: 'bilanciata' | 'quasi-bilanciata' | 'da-correggere'
}
```

### i18n — EN + IT from day 1

Both English and Italian must be fully implemented before shipping — not as TODOs. Use next-intl with the same setup as the main platform:

- Translation files: `messages/en.json` and `messages/it.json`
- Routes: `/en/balancer/new` and `/it/balancer/new` (same pattern as the main site)

- Ingredient names in the database have both `nome` (Italian) and `nameEn` (English) fields — display based on locale
- Parameter names switch language: EN uses "Sugars / Fat / MSNF / Other Solids / Total Solids / POD / PAC" — IT uses "Zuccheri / Grassi / SLNG / Altri Solidi / Solidi Totali / POD / PAC"
- The language toggle [EN | IT] in the header switches the full route just like the main website

### Design — must match the main website exactly

The balancer will be merged into the main gelato education platform. A user who switches from the course player to the balancer must feel zero visual discontinuity. Rules:
- Same CSS custom properties (see Design System section above) — no new color values
- Same fonts: Cormorant Garamond italic for the app title only; Inter for everything else (all data, labels, numbers, table content)
- Same border radius, spacing tokens, button styles, focus rings
- Same dark mode implementation (`[data-theme="dark"]` on html, localStorage persistence, system preference default)
- Same minimum touch target: 44×44px for all interactive elements
- Same focus ring: `outline: 2px solid var(--color-accent); outline-offset: 2px`
- No gradients, no purple/violet/indigo, no decorative blobs, no generic SaaS patterns
- Status indicators use the same green/yellow/red semantic tokens already defined

### Other notes

- The built-in ingredient database is read-only. Users can only add/edit/delete their own custom ingredients.
- All calculations are pure functions in `calculations.ts` — no side effects, easily testable.
- When this merges into the main education platform: the /balancer routes drop into the existing Next.js app, design tokens are already identical, and the only wiring needed is Supabase Auth to replace localStorage for cloud saves.
